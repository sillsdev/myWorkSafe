Customization

myWorkSafe is basically intended to have zero configuration.  I, as an occasional supporter of language workers, do not have time to individually configure their backup programs. Instead, I want myWorkSafe to embody all the same decisions I would make for a typical native-speaker language worker.  This configuration ships with in the form of a “myWorkSafe.ini” file found in the folder where the application was installed.  For example here are some entries which illustrate many of the options:
 

[Paratext]
#on 32-bit Windows
rootFolder=HKEY_LOCAL_MACHINE\Software\ScrChecks\1.0\Settings_Directory

#on 64 bit Windows
rootFolder=HKEY_LOCAL_MACHINE\SOFTWARE\Wow6432Node\ScrChecks\1.0\Settings_Directory

#in case that registry entry is missing
rootFolder=c:\my paratext projects   

excludeFolder=TNE Notes Database
excludeFolder=cms

[AdaptIt]
rootFolder=$Documents$\AdaptIt Unicode Work
#try this one if that one doesn't work
rootFolder=$Documents$\AdaptIt Work

[WeSay]
rootFolder=$MyDocuments$\WeSay
excludeFile=*.pdf

[MyPictures]
name = My Pictures
rootFolder = $MyPictures$
excludeFolder=Sample Pictures

 
If you want, you can make different decisions, or just add to myWorkSafe’s smarts, by modifying the “myWorkSafe.ini” found in your appData/local/myWorkSafe (or whatever the equivalent is on that version of Windows).  A blank version of this file is created after the first use of the program.  So, for example, I’m not a typical language worker, so my settings alter things radically (I basically have too much data to ever fit on a USB key, so I limit myWorkSafe to a few key things):
 
#I never would put something I want to backup on the desktop.
[Desktop]
skip

#My machine is loaded with other people’s data, but this language
#is one we actually work in, so I want to give it priority
[EdoloData]
rootFolder = $MyDocuments$ /LanguageData/Edolo

#I keep my WeSay folder somewhere special

[WeSay]
rootFolder=c:\dev\WeSayProjects


So far, we’ve seen two levels of settings: one comes with the program, and another can override that a user with special needs.  There is a third level: drive-specific settings.  For example, you could have one drive for data, and another for videos.  In my case, I like to get a very small amount of data backed up constantly: my current list of passwords/registrations.  To do that, place a file named, again, “myWorkSafe.ini” inside the folder used for backup on the device; in my case, “f:/myWorkSafe-John-THINKPAD”. Here’s the contents:

[Settings]
# I don't want this to be affected by myWorkSafe’s factory settings, at all
clearAllPreviousSettings

[Passwords]
rootFolder= C:\Users\John\Documents\Passwords

Summary

If you find yourself frequently customizing colleague’s myWorkSafe’s, something is wrong.  The point is to have enough smarts and domain knowledge to make this unnecessary.  But these configuration abilities should be enough to make the program work for some of us geeks, and we need that to happen, so that myWorkSafe gets a lot of testing.
